# 🚀 Deployment Guide

Three deployment paths, ordered easiest → most flexible:

1. [Vercel (frontend) + Render (backend + Postgres)](#path-1-vercel--render-recommended)
2. [Single VPS with Docker Compose](#path-2-single-vps-docker-compose)
3. [Manual systemd / nginx setup](#path-3-manual)

---

## Path 1 — Vercel + Render (recommended)

This combines Vercel's free Next.js hosting with Render's free FastAPI + Postgres tier. Total cost: **$0** for typical demo traffic.

### 1. Push your repo to GitHub
Make sure `main` branch contains everything you want to deploy.

### 2. Provision the backend on Render
1. Sign in at <https://render.com>
2. Click **New ▾ → Blueprint**
3. Connect your GitHub account, pick this repository
4. Render reads [`render.yaml`](render.yaml) and proposes:
   - `interview-prep-backend` (FastAPI web service)
   - `interview-prep-db` (managed Postgres)
5. Click **Apply**. Wait ~3 minutes for the first build.

🟢 You should now see a URL like `https://interview-prep-backend.onrender.com`. Visit `/docs` to confirm.

### 3. Set extra env vars on Render
Render → your service → **Environment**:

| Key | Value |
|---|---|
| `CORS_ORIGINS` | `https://your-frontend.vercel.app` (set after step 4 below) |
| `ADMIN_EMAILS` | your email — auto-promotes to admin on signup (optional) |
| `GROQ_API_KEY` | get a free one at <https://console.groq.com> (optional) |

`JWT_SECRET` is auto-generated by Render. `DATABASE_URL` is auto-injected.

### 4. Deploy the frontend on Vercel
1. Sign in at <https://vercel.com>
2. **Add New ▾ → Project** → import this repo
3. Vercel auto-detects Next.js. Set the root directory to `frontend/`.
4. Under **Environment Variables** add:
   - `NEXT_PUBLIC_API_URL = https://interview-prep-backend.onrender.com`
5. Click **Deploy**.

🟢 Done. Update Render's `CORS_ORIGINS` to your final Vercel URL and **Save** to trigger a redeploy.

### 5. Optional — enable Whisper on Render
Whisper transcription needs `requirements-ml.txt` and `ffmpeg`. Render's free tier doesn't ship `ffmpeg` so:
- Either use a paid tier with the apt package
- Or switch to the **Docker** build by setting Render's runtime to *Docker* (it'll use `backend/Dockerfile` which already installs ffmpeg + ML deps)

## Path 2 — Single VPS (Docker Compose)

Any 2-vCPU / 4 GB Linux VM works (DigitalOcean, Linode, Hetzner, AWS Lightsail).

```bash
# 1. SSH into your VM
ssh root@your-server.example.com

# 2. Install Docker + Compose
curl -fsSL https://get.docker.com | sh

# 3. Clone the repo
git clone https://github.com/yashonaath-06/AI-Powered-Interview-Preparation-Platform.git
cd AI-Powered-Interview-Preparation-Platform

# 4. Create .env
cp .env.example .env
# Edit JWT_SECRET to a long random string

# 5. Start everything
docker compose up -d --build

# 6. Verify
curl http://localhost:8000/api/health
```

For HTTPS, put **Caddy** or **Nginx** in front of `localhost:3000` and `localhost:8000`. Point DNS at your VM and you're done.

## Migrations

The backend currently uses `Base.metadata.create_all` (idempotent — no-op for existing tables). For schema changes between releases, add a column / model and either:

- Reset your dev DB: `rm backend/interview_prep.db`
- Or wire up Alembic (one-time setup, then `alembic revision --autogenerate -m "..."` per change).

## Rolling out a new version

| Service | Mechanism |
|---|---|
| Render backend | Push to `main` → auto-redeploys (`autoDeploy: true` in `render.yaml`) |
| Vercel frontend | Push to `main` → auto-deploys preview, promote to production |
| Single VPS | `git pull && docker compose up -d --build` |

## Smoke-testing a fresh deployment

Run these against your production URL:

```bash
curl https://your-backend/api/health
# → {"status": "ok"}

curl -X POST https://your-backend/api/auth/signup \
  -H 'Content-Type: application/json' \
  -d '{"email":"smoke@test.com","full_name":"Smoke Test","password":"smoke12345"}'
# → 201 with access_token

curl https://your-frontend
# → renders the landing page HTML
```

If anything fails, check Render logs (`Logs` tab) and Vercel logs (`Deployments → click run → Logs`).
